Render Hardware Interface（RHI）简单来说就是一个抽象层，它位于游戏引擎或图形应用和底层图形API（比如DirectX、Vulkan、Metal等）之间，主要作用就是让开发者不用直接去碰那些复杂又容易出错的底层API，而是通过一个统一的接口去操作GPU，同时可以更方便地支持跨平台运行。这里主要参考 nvrhi 来构建一个支持 dx12 和 vulkan 的 RHI.

根据图形管线的每个阶段, 我们分别建立相应的接口或数据结构.

Input Assembler (Vertex Input)

由于在不同的情况下, 我们需要向 Vertex Shader 提交不同类型的自定义 Vertex 结构体, 因此需要向 pipeline 描述自定义的顶点格式.

  • name: DX12 中需要指定与顶点结构体元素相关联的字符串, 为 D3D12_INPUT_ELEMENT_DESC::SemanticName.

    • // 如当在 cpu 端指定 position 元素的 Semantic Name 为 POSITION.
// gpu 端应表示如下以供关联.

struct VertexInput
{
    float3 position : POSITION;
};

  • element_stride: 整个顶点结构体的大小, 一般即为 sizeof(Vertex).

  • array_size: 在 DX12 中, 其指定 D3D12_INPUT_ELEMENT_DESC::SemanticIndex, 而在 Vulkan 的设计中, 则直接依次排开 (具体看后面如何初始化 InputLayout).

    • // 有些时候顶点不止有一套 texcoord, 就需要用到 大于 1 的 array_size.

// 当 cpu 端指定 Vertex 和 VertexAttribute 如下:
struct Vertex
{
    float3 position;
    float3 normal;
    float2 uv0;
    float2 uv1;
    float2 uv2;
};

attribute[0].name = "POSITION";
attribute[0].offset = offsetof(Vertex, position);
attribute[2].array_size = 1;
...
attribute[1].name = "NORMAL";
attribute[1].offset = offsetof(Vertex, normal);
attribute[1].array_size = 1;
...
attribute[2].name = "TEXCOORD";
attribute[2].offset = offsetof(Vertex, uv0);
attribute[2].array_size = 3;
...

// 其在 GPU 端需要如下指定:
struct VertexInput
{
    float3 position : POSITION;
    float3 normal : NORMAL;
    float2 uv0 : TEXCOORD0;    // SemanticIndex 为 0.
    float2 uv1 : TEXCOORD1;    // SemanticIndex 为 1.
    float2 uv2 : TEXCOORD2;    // SemanticIndex 为 2.
};
DX12

其 initialize() 的思路是, 将 VertexAttributeDescArray 复制到 _vertex_attribute_descs 中, 并根据其数据填充 _d3d12_input_element_descs.
